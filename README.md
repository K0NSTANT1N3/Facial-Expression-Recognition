# ექსპერიმენტი 1 — საბაზისო CNN მოდელი

## მიზანი

პირველ ეტაპზე ჩემი მიზანი იყო მაქსიმალურად მარტივი ნეირონული ქსელის აგება, რომელიც საბაზისო წერტილად გამომადგებოდა შემდგომი ექსპერიმენტებისთვის. ამ ეტაპზე მთავარი მიზანი არ ყოფილა საუკეთესო შედეგის მიღება. პირიქით, მსურდა შემექმნა მარტივი არქიტექტურა, რომლის ქცევასაც ადვილად დავაკვირდებოდი და რომლის საფუძველზეც შესაძლებელი იქნებოდა შემდგომი გაუმჯობესებების შეფასება.

ნებისმიერი უფრო რთული მოდელის დამატება მხოლოდ მაშინ არის აზრიანი, როდესაც არსებობს საწყისი შედეგი, რომელთანაც შედარება შეიძლება. სწორედ ამიტომ ექსპერიმენტები დავიწყე მცირე ზომის CNN არქიტექტურით.

---

ამ არქიტექტურის არჩევის მთავარი მიზეზი იყო მისი სიმარტივე. ორი კონვოლუციური ფენა საკმარისია იმის შესამოწმებლად, შეუძლია თუ არა მოდელს სახის ძირითადი მახასიათებლების სწავლა, ხოლო მცირე რაოდენობის პარამეტრები ამარტივებს მოდელის ქცევის ანალიზს.

## გამოყენებული ჰიპერპარამეტრები

| პარამეტრი     | მნიშვნელობა      |
| ------------- | ---------------- |
| Optimizer     | Adam             |
| Learning Rate | 0.001            |
| Batch Size    | 64               |
| Epochs        | 10               |
| Loss Function | CrossEntropyLoss |

Adam ოპტიმიზატორი შევარჩიე, რადგან პრაქტიკაში ერთ-ერთ ყველაზე სტაბილურ და ფართოდ გამოყენებულ ოპტიმიზატორად ითვლება და საწყისი ექსპერიმენტებისთვის კარგი არჩევანია.

Learning Rate = 0.001 ავირჩიე, რადგან ეს მნიშვნელობა ხშირად გამოიყენება საწყის წერტილად CNN მოდელების ტრენინგისას.

CrossEntropyLoss გამოყენებულია იმის გამო, რომ FER2013 წარმოადგენს მრავალკლასიან კლასიფიკაციის ამოცანას, სადაც თითოეულ სურათს ზუსტად ერთი სწორი კლასი შეესაბამება.

Batch Size = 64 შეირჩა გამოთვლითი რესურსებისა და ტრენინგის სტაბილურობის ბალანსის შესანარჩუნებლად.

## შედეგების ანალიზი

ტრენინგის პროცესში აშკარად ჩანს, რომ მოდელი სწავლობს. Training Accuracy სტაბილურად იზრდება ყველა ეპოქაში, ხოლო Training Loss მუდმივად მცირდება. ეს მიუთითებს, რომ მოდელს შეუძლია სასარგებლო პატერნების ამოცნობა სასწავლო მონაცემებში.

Validation Accuracy-ს ზრდაც შეინიშნება საწყის ეპოქებში, თუმცა დაახლოებით მერვე ეპოქის შემდეგ გაუმჯობესება პრაქტიკულად ჩერდება. ამავდროულად, Training Accuracy აგრძელებს ზრდას, ხოლო Validation Loss იწყებს ზრდას.

ეს ქცევა მიუთითებს Overfitting-ის საწყის ნიშნებზე. მოდელი სულ უფრო უკეთ სწავლობს Training მონაცემებს, თუმცა მიღებული ცოდნის განზოგადება Validation მონაცემებზე უკვე რთულდება.

ამის დასტურია Training Accuracy-სა და Validation Accuracy-ს შორის განსხვავებაც. საბოლოო ეპოქაზე Training Accuracy აღწევს 64.9%-ს, მაშინ როდესაც Validation Accuracy მხოლოდ 51.8%-ს შეადგენს.

## დასკვნა

საბაზისო ექსპერიმენტმა დაადასტურა, რომ ტრენინგის pipeline სწორად მუშაობს და მარტივი CNN მოდელს შეუძლია FER2013 მონაცემებზე მნიშვნელოვანი მახასიათებლების სწავლა.

მიუხედავად ამისა, მიღებული შედეგები აჩვენებს, რომ მოდელს აქვს გარკვეული შეზღუდვები. ერთი მხრივ, შესაძლებელია არქიტექტურა ზედმეტად მარტივი იყოს უფრო რთული სახის მახასიათებლების დასასწავლად. მეორე მხრივ, უკვე შეინიშნება Overfitting-ის ნიშნები, რაც რეგულარიზაციის მეთოდების გამოყენების საჭიროებაზე მიუთითებს.

შემდგომ ექსპერიმენტებში ვგეგმავ:

* დამატებითი კონვოლუციური ფენების დამატებას;
* Batch Normalization-ის გამოყენებას;
* Dropout-ის დამატებას;
* სხვადასხვა ჰიპერპარამეტრების შედარებას;
* მიღებული შედეგების W&B-ზე დეტალურ ანალიზს.

ეს მოდელი გამოყენებული იქნება როგორც საწყისი საბაზისო არქიტექტურა, რომელთანაც ყველა შემდგომი ექსპერიმენტის შედეგები შედარდება.





# ექსპერიმენტი 2 — CNN არქიტექტურის გაღრმავება

## მოტივაცია

საბაზისო მოდელის შედეგების ანალიზის შემდეგ გაჩნდა ჰიპოთეზა, რომ მოდელს შესაძლოა არ ჰქონდა საკმარისი შესაძლებლობა (capacity), რათა ესწავლა უფრო რთული სახის მახასიათებლები. მიუხედავად იმისა, რომ მოდელი სწავლობდა და Validation Accuracy იზრდებოდა, საბოლოო შედეგი დაახლოებით 52%-ის ფარგლებში დარჩა.

ამ ჰიპოთეზის შესამოწმებლად გადავწყვიტე ეტაპობრივად გამეღრმავებინა CNN არქიტექტურა და შემეფასებინა დამატებითი კონვოლუციური ფენების გავლენა მოდელის წარმადობაზე.

### ცვლილება

საბაზისო არქიტექტურას დაემატა მესამე კონვოლუციური ბლოკი.

Validation Accuracy გაიზარდა 51.8%-დან 53.2%-მდე, ხოლო Train Accuracy 69.6%-მდე ავიდა.

მიღებული შედეგი მიუთითებს, რომ დამატებითმა კონვოლუციურმა ფენამ მოდელს საშუალება მისცა ესწავლა უფრო კომპლექსური ვიზუალური მახასიათებლები. შესაბამისად, შეიძლება ითქვას, რომ საბაზისო მოდელს მართლაც აკლდა გარკვეული capacity.

## ექსპერიმენტი 2.2 — ოთხი კონვოლუციური ბლოკი

### ცვლილება

შემდეგ ეტაპზე მოდელს კიდევ ერთი კონვოლუციური ბლოკი დაემატა, რათა შემემოწმებინა გაგრძელდებოდა თუ არა შედეგების გაუმჯობესება არქიტექტურის შემდგომი გაღრმავების შემთხვევაში.

### შედეგები

| მეტრიკა | მნიშვნელობა |
|----------|----------|

### ანალიზი

მიუხედავად იმისა, რომ მოდელი წარმატებით სწავლობდა Training მონაცემებზე, Validation Accuracy პრაქტიკულად აღარ გაუმჯობესებულა და 52.9%-ზე დარჩა.

ეს შედეგი აჩვენებს, რომ არქიტექტურის დამატებითი გაღრმავება ავტომატურად არ იწვევს უკეთეს გენერალიზაციას. მოცემულ შემთხვევაში მეოთხე კონვოლუციურმა ბლოკმა ვერ მოახდინა მნიშვნელოვანი დადებითი გავლენა Validation შედეგებზე.

## დასკვნა

ჩატარებული ექსპერიმენტებიდან საუკეთესო შედეგი აჩვენა სამკონვოლუციურმა CNN მოდელმა, რომელმაც მიაღწია 53.2% Validation Accuracy-ს.

ექსპერიმენტების საფუძველზე შეიძლება დავასკვნათ, რომ:

- საბაზისო მოდელს მართლაც აკლდა გარკვეული capacity;
- არქიტექტურის ზომიერი გაღრმავება სასარგებლო აღმოჩნდა;
- დამატებითი სიღრმე ავტომატურად არ იწვევს უკეთეს შედეგს;
- მოცემულ ეტაპზე უფრო მიზანშეწონილია ტრენინგის სტაბილიზაციისა და რეგულარიზაციის მეთოდების გამოკვლევა, ვიდრე CNN ფენების რაოდენობის უსასრულოდ გაზრდა.

ტრენინგის პროცესში Training Accuracy მუდმივად იზრდებოდა და საბოლოოდ 90.5%-ს მიაღწია. ამავე დროს Training Loss სტაბილურად მცირდებოდა, რაც მიუთითებს, რომ მოდელი წარმატებით სწავლობდა Training მონაცემებს.

Validation მონაცემებზე განსხვავებული სურათი მივიღეთ. საწყის ეპოქებში Validation Accuracy იზრდებოდა, თუმცა დაახლოებით მე-7 და მე-14 Epoch-ის შემდეგ გაუმჯობესება პრაქტიკულად შეწყდა და შედეგი 52–53%-ის ფარგლებში დარჩა.

განსაკუთრებით საინტერესო აღმოჩნდა Validation Loss-ის ქცევა. მიუხედავად იმისა, რომ Training Loss მუდმივად მცირდებოდა, Validation Loss მნიშვნელოვნად იზრდებოდა. ეს მიუთითებს, რომ მოდელი სულ უფრო მეტად ერგებოდა Training მონაცემებს და კარგავდა გენერალიზაციის უნარს.

---

### Overfitting-ის ინტერპრეტაცია

მიღებული შედეგები წარმოადგენს Overfitting-ის კლასიკურ მაგალითს.

ტრენინგის საწყის ეტაპზე მოდელი სწავლობს ზოგად მახასიათებლებს, რომლებიც სასარგებლოა როგორც Training, ასევე Validation მონაცემებისთვის. მაგალითად:

- სახის ძირითადი ფორმები;
- თვალების მდებარეობა;
- პირის მოძრაობა;
- ემოციებისთვის დამახასიათებელი საერთო ვიზუალური ნიშნები.

მოგვიანებით მოდელი იწყებს Training მონაცემების სპეციფიკური დეტალების დამახსოვრებას. შედეგად:

- Training Accuracy აგრძელებს ზრდას;
- Validation Accuracy პრაქტიკულად აღარ უმჯობესდება;
- Validation Loss იზრდება.

ეს ნიშნავს, რომ მოდელი უფრო და უფრო კარგად სწავლობს სასწავლო მონაცემებს, თუმცა მიღებული ცოდნის განზოგადება ახალ მონაცემებზე აღარ ხდება.

---

### რატომ იზრდება Validation Loss?

Validation Accuracy დაახლოებით უცვლელი რჩება, თუმცა Validation Loss მკვეთრად იზრდება.

ეს შეიძლება აიხსნას იმით, რომ მოდელი Validation მონაცემებზე სულ უფრო თავდაჯერებულ არასწორ პროგნოზებს აკეთებს.

მაგალითად, თუ საწყის ეტაპზე მოდელი არასწორ კლასს 40%-იანი ალბათობით ანიჭებდა, მოგვიანებით შესაძლოა იგივე შეცდომა 95%-იანი დარწმუნებით გააკეთოს.

ასეთ შემთხვევაში Accuracy თითქმის უცვლელი დარჩება, თუმცა Loss მნიშვნელოვნად გაიზრდება.

---

### დასკვნა

ამ ექსპერიმენტმა აჩვენა, რომ ოთხკონვოლუციურ CNN-ს საკმარისზე მეტი შესაძლებლობა (capacity) აქვს Training მონაცემების დასასწავლად.

მიუხედავად იმისა, რომ Training Accuracy 90%-ს გადააჭარბა, Validation Accuracy დაახლოებით 53%-ის ფარგლებში დარჩა. ეს მიუთითებს, რომ მოდელის მთავარი პრობლემა უკვე აღარ არის არასაკმარისი სიღრმე ან capacity.

პირიქით, მიღებული შედეგები აჩვენებს, რომ არქიტექტურის შემდგომი გაღრმავება ნაკლებად პერსპექტიულია და ამ ეტაპზე უფრო მნიშვნელოვანია Overfitting-ის შემცირება და გენერალიზაციის გაუმჯობესება.

ამიტომ შემდეგ ექსპერიმენტებში ყურადღება გამახვილდება:

- Batch Normalization-ის დამატებაზე;
- Dropout-ის გამოყენებაზე;
- რეგულარიზაციის მეთოდების ტესტირებაზე;
- სხვადასხვა ჰიპერპარამეტრების შედარებაზე.

მიღებული შედეგების საფუძველზე შეიძლება დავასკვნათ, რომ მოდელის სიღრმის ზრდამ გარკვეულ ეტაპამდე დადებითი გავლენა მოახდინა, თუმცა ოთხი კონვოლუციური ფენის შემდეგ დამატებითი სიღრმე უკვე აღარ იძლევა მნიშვნელოვან გაუმჯობესებას და მოდელი იწყებს Training მონაცემებზე გადაჭარბებულ მორგებას.


# ექსპერიმენტი 3 — Batch Normalization

## მოტივაცია

წინა ექსპერიმენტების შედეგად დადგინდა, რომ არქიტექტურის ზომიერი გაღრმავება სასარგებლო იყო, თუმცა Validation Accuracy კვლავ 53%-ის ფარგლებში რჩებოდა.

ამ ეტაპზე გაჩნდა ჰიპოთეზა, რომ მოდელის სწავლა შესაძლოა არ იყო საკმარისად სტაბილური და კონვოლუციური ფენების აქტივაციები ტრენინგის პროცესში მნიშვნელოვნად იცვლებოდა.

ამ პრობლემის გადასაჭრელად მოდელს დაემატა Batch Normalization ფენები.

---

## განხორციელებული ცვლილებები

Batch Normalization დაემატა:

- თითოეული Conv2D ფენის შემდეგ;
- სრულად დაკავშირებულ (Fully Connected) ნაწილში Linear ფენის შემდეგ.

ახალი ბლოკის სტრუქტურა გახდა:

```text
Conv2D
↓
BatchNorm
↓
ReLU
↓
MaxPool
```

ხოლო კლასიფიკატორში:

```text
Linear
↓
BatchNorm
↓
ReLU
↓
Output
```

---

## Batch Normalization-ს მუღამი

ტრენინგის პროცესში თითოეული ფენის აქტივაციების განაწილება მუდმივად იცვლება, რადგან წინა ფენების წონები ყოველ Batch-ზე ახლდება.

Batch Normalization ამ აქტივაციებს ნორმალიზებას უკეთებს და მათ საშუალოს 0-თან, ხოლო ვარიაციას 1-თან აახლოებს.

ამის შედეგად:

- ტრენინგი ხდება უფრო სტაბილური;
- გრადიენტების გავრცელება უმჯობესდება;
- შესაძლებელია უფრო სწრაფი კონვერგენცია;
- ზოგიერთ შემთხვევაში მცირდება Overfitting-იც.

---

## შედეგები

| მეტრიკა | მნიშვნელობა |
|----------|----------|
| Final Train Accuracy | 98.5% |
| Final Validation Accuracy | 54.3% |
| Best Validation Accuracy | 54.9% |

---

## ანალიზი

Validation Accuracy გაიზარდა დაახლოებით 53.2%-დან 54.9%-მდე.

ეს მიუთითებს, რომ Batch Normalization-მა მოდელს საშუალება მისცა უფრო ეფექტურად ესწავლა მონაცემებიდან სასარგებლო მახასიათებლები.

განსაკუთრებით შესამჩნევია ტრენინგის სიჩქარე.

უკვე მესამე Epoch-ზე Validation Accuracy 54%-ს აჭარბებს, რაც წინა ექსპერიმენტებში ბევრად გვიან ხდებოდა.

ეს ადასტურებს, რომ Batch Normalization აუმჯობესებს ოპტიმიზაციის პროცესს და მოდელი უფრო სწრაფად კონვერგირდება.

---

## Overfitting-ის ანალიზი

მიუხედავად Validation Accuracy-ის გაუმჯობესებისა, კვლავ შეინიშნება ძლიერი Overfitting.

საბოლოო ეპოქაზე:

```text
Train Accuracy = 98.5%
Validation Accuracy = 54.3%
```

Training Accuracy თითქმის სრულყოფილ შედეგს აღწევს, ხოლო Validation Accuracy მნიშვნელოვნად ჩამორჩება.

ამავე დროს Validation Loss დროთა განმავლობაში იზრდება:

```text
Epoch 3  -> 1.21
Epoch 10 -> 1.95
Epoch 20 -> 2.50
```

ეს მიუთითებს, რომ მოდელი სულ უფრო მეტად იმახსოვრებს Training მონაცემებს და ვერ ახერხებს მიღებული ცოდნის ეფექტურად განზოგადებას ახალ მონაცემებზე.

---

## დასკვნა

Batch Normalization-ის დამატება წარმატებული ცვლილება აღმოჩნდა.

ექსპერიმენტმა აჩვენა, რომ:

- ტრენინგი უფრო სწრაფი და სტაბილური გახდა;
- Validation Accuracy გაუმჯობესდა;
- მოდელმა უკეთ ისწავლა მონაცემების წარმოდგენა.

თუმცა Overfitting კვლავ მნიშვნელოვან პრობლემად რჩება.

ამიტომ შემდეგი ექსპერიმენტის მთავარი მიზანი იქნება გენერალიზაციის გაუმჯობესება და Overfitting-ის შემცირება Dropout-ის გამოყენებით.


# ექსპერიმენტი 4 — Batch Normalization + Dropout

## მოტივაცია

წინა ექსპერიმენტში Batch Normalization-ის დამატებამ Validation Accuracy დაახლოებით 54.9%-მდე გაზარდა. მიუხედავად ამისა, Training Accuracy 98%-ს აღემატებოდა, ხოლო Validation Accuracy მნიშვნელოვნად ჩამორჩებოდა.

ეს მიუთითებდა ძლიერ Overfitting-ზე. მოდელი ძალიან კარგად სწავლობდა Training მონაცემებს, თუმცა ვერ ახერხებდა მიღებული ცოდნის ეფექტურად განზოგადებას ახალ მაგალითებზე.

ამ პრობლემის შესამცირებლად Batch Normalization-იან არქიტექტურას დაემატა Dropout რეგულარიზაცია.

---

## განხორციელებული ცვლილება

კლასიფიკატორის ნაწილში დაემატა:

```text
Dropout(p=0.3)
```

საბოლოო სტრუქტურა გახდა:

```text
Linear
↓
BatchNorm
↓
ReLU
↓
Dropout(0.3)
↓
Output
```

Dropout ტრენინგის დროს შემთხვევით თიშავს ნეირონების ნაწილს, რის შედეგადაც მოდელი ვერ ეყრდნობა კონკრეტულ ნეირონებს და იძულებულია ისწავლოს უფრო განზოგადებადი წარმოდგენები.

---

## შედეგები

| მეტრიკა                   | მნიშვნელობა |
| ------------------------- | ----------- |
| Final Train Accuracy      | 70.2%       |
| Final Validation Accuracy | 57.5%       |
| Best Validation Accuracy  | 57.9%       |

---

## ანალიზი

ეს ექსპერიმენტი აღმოჩნდა ყველაზე წარმატებული ამ ეტაპამდე.

Batch Normalization-ის მქონე მოდელი აღწევდა დაახლოებით 54.9% Validation Accuracy-ს, ხოლო Dropout-ის დამატების შემდეგ შედეგი გაიზარდა 57.9%-მდე.

განსაკუთრებით საინტერესოა Training Accuracy-ის ცვლილება:

| მოდელი              | Train Accuracy | Validation Accuracy |
| ------------------- | -------------- | ------------------- |
| BatchNorm           | 98.5%          | 54.9%               |
| BatchNorm + Dropout | 70.2%          | 57.9%               |

მიუხედავად იმისა, რომ Training Accuracy მნიშვნელოვნად შემცირდა, Validation Accuracy გაიზარდა.

ეს მიუთითებს, რომ მოდელი ნაკლებად იმახსოვრებს Training მონაცემებს და უკეთ სწავლობს ზოგად პატერნებს, რომლებიც ახალ მონაცემებზეც მუშაობს.

---

## რატომ იმუშავა Dropout-მა?

Batch Normalization აუმჯობესებს ტრენინგის სტაბილურობას და აჩქარებს სწავლის პროცესს, თუმცა თავისთავად არ აგვარებს Overfitting-ის პრობლემას.

Dropout ამატებს რეგულარიზაციას. ტრენინგის დროს ნეირონების ნაწილი დროებით ითიშება, რის შედეგადაც მოდელი ვერ ეყრდნობა კონკრეტულ აქტივაციებს და იძულებულია ინფორმაცია სხვადასხვა გზით ისწავლოს.

ამან მნიშვნელოვნად შეამცირა Overfitting და გააუმჯობესა გენერალიზაცია Validation მონაცემებზე.

---

## დასკვნა

ჩატარებულმა ექსპერიმენტმა აჩვენა, რომ მიმდინარე ამოცანისთვის ყველაზე ეფექტური აღმოჩნდა Batch Normalization-ისა და Dropout-ის კომბინაცია.

მოდელმა მიაღწია ამ ეტაპისთვის საუკეთესო შედეგს — 57.9% Validation Accuracy-ს.

მიღებული შედეგები ადასტურებს, რომ მოდელის მთავარი პრობლემა უკვე აღარ იყო capacity-ის ნაკლებობა, არამედ Overfitting. Dropout-ის დამატებამ მნიშვნელოვნად შეამცირა ეს პრობლემა და უზრუნველყო Validation შედეგების შესამჩნევი გაუმჯობესება.

შემდეგ ეტაპზე იგეგმება კლასების დისბალანსის გავლენის შესწავლა Weighted Loss ფუნქციის გამოყენებით და სხვადასხვა ჰიპერპარამეტრების დამატებითი ექსპერიმენტები.

ასეთ სიტუაციაში სტანდარტული Cross Entropy Loss ხშირად უპირატესობას ანიჭებს უფრო ხშირ კლასებს, რადგან მათი შეცდომები მთლიან Loss-ში ბევრად უფრო დიდ გავლენას ახდენს.

ამის შესაძლო მიზეზია, რომ feature map-ის დიდი ზომის შენარჩუნებამ გაზარდა classifier-ის პარამეტრების რაოდენობა და მოდელმა უფრო მეტად დაიწყო Training მონაცემების დამახსოვრება.

შედეგად Training Accuracy მაღალი დარჩა, მაგრამ Validation Accuracy შემცირდა, რაც მიუთითებს უფრო ძლიერ overfitting-ზე.

## ექსპერიმენტი 6 — Data Augmentation

წინა ექსპერიმენტებში გამოჩნდა, რომ Batch Normalization-ისა და Dropout-ის დამატების შემდეგ მოდელმა მიაღწია დაახლოებით 57-58% validation accuracy-ს, თუმცა training accuracy მნიშვნელოვნად აღემატებოდა validation accuracy-ს. ეს მიუთითებდა, რომ მოდელი ნაწილობრივ მაინც სწავლობდა training მონაცემების სპეციფიკურ მახასიათებლებს.

შემდეგი ჰიპოთეზა იყო, რომ მონაცემების ხელოვნური გაზრდა (Data Augmentation) მოდელს დაეხმარებოდა უფრო ზოგადი მახასიათებლების სწავლაში და გააუმჯობესებდა unseen მონაცემებზე მუშაობას.

გამოყენებული იქნა შემდეგი augmentation-ები:

- RandomHorizontalFlip(p=0.5)
- RandomRotation(10)

Augmentation გამოიყენებოდა მხოლოდ training set-ზე, ხოლო validation set უცვლელი დარჩა, რათა შეფასება რეალურ პირობებთან მაქსიმალურად ახლოს ყოფილიყო.

ასევე აღსანიშნავია, რომ augmentation-ის შემდეგ training accuracy მნიშვნელოვნად შემცირდა. ეს მიუთითებს, რომ მოდელს გაცილებით რთული სასწავლო ამოცანა მიეცა და მან ნაკლებად დაიწყო training მონაცემების დამახსოვრება.

# ექსპერიმენტი 8 — ResNet არქიტექტურის გამოყენება და Fine-tuning

წინა ექსპერიმენტებში გამოყენებული მქონდა საკუთარი CNN არქიტექტურები, სადაც ეტაპობრივად ვზრდიდი მოდელის სირთულეს, ვამატებდი Batch Normalization-ს, Dropout-ს და ვცდილობდი overfitting-ის შემცირებას.

შემდეგ ეტაპზე გადავწყვიტე გამომეცადა ResNet არქიტექტურა.

ResNet-ის მთავარი განსხვავება ჩვეულებრივი CNN-ისგან არის **Residual Connection**, რომელიც საშუალებას აძლევს ღრმა ქსელს უკეთ შეინარჩუნოს ინფორმაცია და შეამციროს gradient vanishing პრობლემა.

ჰიპოთეზა იყო:

> უფრო ღრმა და წინასწარ კარგად შემუშავებული არქიტექტურა უკეთ ისწავლის სახის ვიზუალურ მახასიათებლებს და მოახდენს უკეთეს generalization-ს.

- ReLU activation
- MaxPooling
- Fully connected classifier

შემდეგ ეტაპობრივად დაემატა:

- მეტი convolution layer
- Batch Normalization
- Dropout

საუკეთესო CNN ვერსიამ მიაღწია დაახლოებით:

```
Validation Accuracy: ~57%
```

მოდელის ქცევა:

- Training accuracy მნიშვნელოვნად იზრდებოდა
- Validation accuracy გარკვეულ მომენტში ჩერდებოდა
- Training loss აგრძელებდა შემცირებას

ეს მიუთითებდა overfitting-ზე.

Dropout და BatchNorm დაეხმარა generalization-ს, თუმცა მოდელის შესაძლებლობები მაინც შეზღუდული დარჩა არქიტექტურის გამო.
